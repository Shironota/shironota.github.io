UML类图

发表于 2018-12-09 | 分类于 UML

概念

类图模拟系统的静态结构，它显示了类，对象，属性和操作之间的关系。

元素

类(Class)

类表示具有共同特征的实体的抽象。 类图主要是用来显示系统中的类、接口以及它们之间的静态结构和关系的一种静态模型。

类图的3个基本组件：类名、属性、方法
用分隔的矩形表示，将类的名称放在第一个分区（居中，粗体，大写），第二个分区列出属性（左对齐，不粗体，小写），第三个分区写入操作（方法）。

接口

Visibility

根据访问修饰符（visibility），类具有不同的访问级别

Marker Visibility
+ public(公有)
- private(私有)
# protected(保护，对子类可见)
~ package(对包内其他成员可见)
/ derived(推导，由其他属性推导得出，不需要直接给定其值)

约束

用来表示各种非局部的关系，如关联路径上的限制

表示方法：将约束放在花括号{}内

类间关系

泛化/继承(Generalization)

一种继承关系，一般与特殊、父类与子类的关系。是对象之间耦合度最大的一种关系

表示方法：带空心三角形的实线，从子类指向父类

实现(Realization)

接口与接口之间可以有类与类之间类似的继承和依赖关系。同时接口与类之间还存在一种实现（Realization）关系，在这种关系中，类实现了接口中声明的方法

表示方法：带空心三角形的虚线，从实现类指向接口

依赖(Dependency)

最弱的一种关联方式，临时性的关联

表示方法：带箭头的虚线，从使用类指向被依赖的类

关联(Associations)

关联表示类之间的静态关系。关联关系使一个类知道另外一个类的属性和方法，关联的两个对象在生命期问题上没有任何约束，可随时解除或关联；关联是共享的，被关联的对象还可以再被其他对象关联。关联又分为一般关联、聚合关联和组合关联。

表示方法：带箭头的实线，箭头从使用类指向被关联的类

形式：

  • 单向关联
  • 双向关联
  • 自关联
  • 多重性关联(Multiplicity)

    在关联的末尾放置多重性符号，表示链接到另一个类的一个实例的类的实例数。就是代表有多少个关联对象存在。

符号 含义
0..1 zero or one
1 one only
0..* 0 or more
1..* 或 * 1 or more
n only n ( n > 1 )
0..n zero to n ( n > 1 )
1..n one to n ( n > 1 )

聚合(Aggregation)

一种不稳定的包含关系，较强于一般关联，有整体和局部的关系，整体不存在时局部可以单独存在。eg：蚂蚁与蚁群，蚂蚁离开蚁群可以单独存在。

表示方法：空心菱形，从局部指向整体

组合(Composition)

一种强烈的包含关系，局部不能脱离整体而存在，一旦整体对象不存在了，局部也随之消亡。eg：部门不能脱离公司而存在。

表示方法：实心菱形，从局部指向整体
