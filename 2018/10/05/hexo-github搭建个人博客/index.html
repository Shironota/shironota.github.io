hexo+github搭建个人博客

发表于 2018-10-05 | 分类于 Hexo

记录搭建hexo博客网站的详细过程

一、环境配置

  • Git
    关于windows下git的下载安装，可以选择直接官网下载git，并使用git for windows自带的git bash来完成相关git命令的执行。也可以先下载安装msys2，利用msys2提供的bash完成git的安装以及相关git命令的执行。msys2提供的bash相比于git bash而言更棒。
    git相关配置问题今后将会另外总结……

  • Nodejs
    官网下载好nodejs后安装即可。然后是配置nodejs。
    找到_C:\Program Files\nodejs\node_modules\npm_目录（具体是你自己的安装路径）下的npmrc文件，打开并编辑成如下：

  • hexo

  • github
    首先你要有一个github账号，然后创建repository，repository的名字是username.github.io(username是你账号的username，这个仓库名字就是搭建成功后博客的域名，所以尽量不要有一些奇怪的username)。

二、搭建博客

    提示成功后，发现选择的目录下多出了一个myblog文件夹，并且配置好了hexo的相关文件。

  3. 本地查看博客页面

    然后访问 http://localhost:4000/ 就能看到个人博客的页面了。

三、本地git项目与github联动

SSH：简单来说就是一种加密的网络协议(Secure Shell)
Git是分布式的管理工具，而远程的代码管理是基于SSH的

  1. 检查本机SSH Key的设置

    显示“No such file or directory”，则进行下一步

  2. 生成新的SSH Key

    期间按一次回车，输入两次密码即可。
    设置的密码是提交项目时的密码，防止别人往项目里乱提交东西。

  3. 将SSH Key添加到Github
    找到id_rsa.pub文件，打开后复制文件里的内容。
    打开github主页，找到Settings，选择SSH and GPG keys，New SSH key,Title随便取一个名字(如：blog),Key中粘贴刚才文件里的内容，Add SSH key。

  4. 测试

    期间输入yes回车，输入之前设置的密码回车。

  • 博客配置文件_config.yml

  • 推送到github

    这个操作是为了将我们写的文章部署到github服务器上并可以被别人浏览。

  • 发布

    期间会弹出github的登录窗口，输入登录即可。

  • OK
    到此，我们的个人博客就算是搭建好了，访问 https://username.github.io/ 就能看到属于自己的博客空间了~
