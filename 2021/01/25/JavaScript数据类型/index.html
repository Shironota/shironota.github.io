JavaScript数据类型

发表于 2021-01-25 | 分类于 JavaScript
字数统计: 5.6k 字 | 阅读时长 ≈ 23 分钟

本篇文章记录JavaScript中的各种数据类型，包括重要的基本概念和常用方法，还有一些容易迷惑的点

数据类型

  • 数值（number）
  • 字符串（string）
  • 布尔值（boolean）
  • 未定义（undefined）
  • 空值（null）
  • 对象（object）
    • 狭义的对象（object）
    • 数组（array）
    • 函数（function）：函数当作数据类型可以赋值给变量，为JS的“函数式编程”奠定基础
  • Symbol（ES6新增）

确定数据类型的方式

JavaScript 有三种方法，可以确定一个值到底是什么类型

  • typeof运算符
  • instanceof运算符
  • Object.prototype.toString方法

typeof运算符

null的类型是object，这是由于历史原因造成的。1995年的 JavaScript 语言第一版，只设计了五种数据类型（对象、整数、浮点数、字符串和布尔值），没考虑null，只把它当作object的一种特殊值。后来null独立出来，作为一种单独的数据类型，为了兼容以前的代码，typeof null返回object就没法改变了

null和undefined

null：可以将其赋值给变量以表示无值（或者叫做值为null），转为数值为0

undefined：表示已声明变量但尚未分配值，转为数值为NaN

布尔值（boolean）

布尔值的转换规则：以下为false，其他视为true

  • undefined
  • null
  • false
  • 0
  • NaN
  • ""或''

特别注意：空数组（[]）和空对象（{}）对应的布尔值都是true

数值（number）

整数和浮点数

JavaScript底层所有的数字都是以64位浮点数形式存储的，故不存在整数的概念（在某些只有整数才能参与的运算中，JavaScript会自动把64位浮点数转换成32位整数）

浮点数不精确

数值精度

IEEE 754浮点数相关问题暂搁置，后续详细学习后再总结这方面知识

数值范围

如果一个数大于等于2的1024次方，那么就会发生“正向溢出”，即 JavaScript 无法表示这么大的数，这时就会返回Infinity

如果一个数小于等于2的-1075次方（指数部分最小值-1023，再加上小数部分的52位），那么就会发生“负向溢出”，即 JavaScript 无法表示这么小的数，这时会直接返回0

JavaScript 提供Number对象的MAX_VALUE和MIN_VALUE属性，返回可以表示的具体的最大值和最小值

科学计数法

以下两种情况，JavaScript会自动将数值转为科学计数法表示

  1. 小数点前的数字多于21位
  2. 小数点后的零多于5个

特殊数值

正零和负零

正零和负零基本上与普通的0一致，唯一的区别是做分母时

除以正零得到+Infinity，除以负零得到-Infinity，这两者是不相等的

NaN

“非数字”（Not a Number）

将字符串解析成数字出错会得到NaN

0/0得到NaN

一些错误的数学运算也会出现NaN

NaN只是Number中的特殊数值，它的数据类型仍是number

NaN不等于任何值（包括它本身）

NaN与任何数（包括它本身）的运算，得到的都是NaN

Infinity

等价于数学意义上的无穷，分正负

特殊的：

Infinity与NaN比较，总是返回false

0乘以Infinity等于NaN

Infinity减去或除以Infinity，等于NaN

Infinity与null计算时，null会转成0，等同于与0的计算

Infinity与undefined计算，返回的都是NaN

数值类型相关的方法

parseInt()

用法：将字符串转为整数

若字符串首部有空格，会自动去掉空格

若传递的参数不是字符串，会先转为字符串再转为整数

js是解释型语言，一个个字符依次转换，遇到不能转换的就直接结束，并返回已转换部分

如果字符串的第一个字符不能转化为数字（后面跟着数字的正负号除外），返回NaN

对于那些会自动转为科学计数法的数字，parseInt会将科学计数法的表示方法视为字符串，因此导致一些奇怪的结果

parseInt方法还可以接受第二个参数（2到36之间），表示被解析值的进制，默认是10

如果第二个参数不是数值，会被自动转为一个整数。这个整数只有在2到36之间，才能得到有意义的结果，超出这个范围，则返回NaN。如果第二个参数是0、undefined和null，则直接忽略

如果字符串包含对于指定进制无意义的字符，则从最高位开始，只返回可以转换的数值。如果最高位无法转换，则直接返回NaN

parseFloat()

将一个字符串转为浮点数，用法与parseInt()类似

注意点：

如果参数不是字符串，或者字符串的第一个字符不能转化为浮点数，则返回NaN

空字符串转换为NaN

区别于Number()函数

Number(object) 函数把对象的值转换为数字

如果对象的值无法转换为数字，那么 Number() 函数返回 NaN

isNaN()

判断一个值是否为NaN

注意：isNaN()仅对传入的参数为数值有效，否则会先转成数值。比如传入字符串、对象、数组时，会被先转成NaN，然后返回true。这一点也说明，isNaN()为true的不一定是NaN

但是，对于空数组和只有一个数值成员的数组，isNaN()返回false

（这里可能不太理解为什么，isNaN()返回false，说明这些参数被Number()解析成数值而不是NaN，涉及到JavaScript中的数据类型转换相关知识，后续待补充…)

上述可以看出，isNaN()并不能严格意义上判断出是否是NaN，若要严格判断，可利用NaN为唯一不等于自身的值的特点

isFinite()

用法：判断传入的参数是否为有限数值

被认为是“无限”的数值：Infinity、-Infinity、NaN、undefined

注意：null被认为是有限的数值

字符串（string）

没有什么特殊的点，JS中的字符串也是不可变的

JavaScript中的原始值（null、undefined、数值、布尔值、字符串）的值都是不可改变的

字符集

JavaScript中使用的是Unicode字符集，允许在程序中使用Unicode码点（\uxxxx）表示字符

在JavaScript中，所有的string类型都是使用UTF-16编码的

对于码点在U+10000到U+10FFFF之间的字符，JavaScript 总是认为它们是两个字符（length属性为2）。也就是说，JavaScript 返回的字符串长度可能是不正确的

关于字符集和字符编码问题不太懂，有时间学习下再总结…

Base64转码

Base64是一种基于64个可打印字符来表示二进制数据的表示方法

64个可打印字符：0～9、A～Z、a～z、+ 和 /

Base64常用于在通常处理文本数据的场合，表示、传输、存储一些二进制数据，包括MIME的电子邮件及XML的一些复杂数据

注意的是，Base64不是加密编码，而是为了不出现特殊字符，处理复杂数据从而简化程序处理

JavaScript原生提供的两个Base64相关方法：

  1. btoa()：任意值转为Base64编码
  2. atob()：Base64编码转为原来的值

注意：上述两个方法不适用于非ASCII码字符（需转码）

对象（object）

创建与属性

基本

对象的键名都是字符串。若设置为数值，则会转成字符串。若不是数值也不符合标识的命名条件，则需加上引号，否则会报错

提取对象的属性有两种方法：点运算符和方括号运算符

注意：

  • 用方括号运算符时[]内要用引号
  • 对于对象的数值键名，不能用点运算符，会被当成小数点而报错，只能用方括号（不太理解键名设置为数值的意义…）

对象的属性可以是任意的数据类型，属性还可动态创建

属性的相关方法

Object.keys()

查看对象的所有属性

delete

删除对象的属性，返回值为true或false

注意点：

  • delete删除一个本就不存在的属性时，也返回true。故不能根据delete结果来判断某个属性是否存在

  • delete只能删除对象本身的属性，无法删除继承的属性。即使执行结果返回true，但实际上这个属性还能使用，并没有真的删除。故不能根据delete结果来判断某个属性是否已被删除

  • 删除一个不能被删除的已存在属性时，返回false

    Object.defineProperty()方法会直接在一个对象上定义一个新属性，或者修改一个对象的现有属性，并返回此对象

in

检查对象中是否包含某个属性

检查的是键名，不是键值

能检查出继承的属性

hasOwnProperty()

判断某个属性是否为对象自身的属性

for…in

遍历一个对象的全部属性

注意：只会遍历出可以遍历的属性，例如继承的toString属性是不可遍历的

with

操作同一个对象的多个属性时，提供一些书写的便利

弊端：绑定对象不明确，无法判断x是全局变量还是对象的属性。这非常不利于代码的除错和模块化，编译器也无法对这段代码进行优化，只能留到运行时判断，这就拖慢了运行速度，建议不使用with语句

引用与值拷贝

对象的引用相关概念与Java类似

JavaScript中原始数据类型（数值、布尔值、字符串、null、undefined）不存在引用的概念，是值拷贝

第一等公民

JavaScript 语言将函数看作一种值，与其它值（数值、字符串、布尔值等等）地位相同。凡是可以使用值的地方，就能使用函数。比如，可以把函数赋值给变量和对象的属性，也可以当作参数传入其他函数，或者作为函数的结果返回。函数只是一个可以执行的值，此外并无特殊之处

由于函数与其他数据类型地位平等，所以在 JavaScript 语言中又称函数为第一等公民

函数的属性和方法

name属性

返回函数的名字

length属性

返回函数预期传入的参数个数

length属性与实际传入的参数个数无关，只反映函数预期传入的参数个数

length属性提供了一种机制，判断定义时和调用时参数的差异，以便实现面向对象编程的“方法重载”（overload）

toString()方法

返回一个字符串，内容是函数的源码，包含内部的注释信息

对于那些原生的函数，toString()方法返回function (){[native code]}

函数作用域

ES5标准，有两种函数作用域：全局作用域、函数作用域（ES6中新增块级作用域）

注意函数作用域只有在函数内部，条件判断语句等内部不算函数作用域，被视作全局变量

函数内部的变量提升

var命令声明的变量，不管在什么位置，变量声明都会被提升到函数体的头部

函数本身的作用域

函数也是一种数据类型，可视作变量，也有自身的作用域

函数执行时所在的作用域，是定义时的作用域，而不是调用时所在的作用域

上面代码中，函数x是在函数f的外部声明的，所以它的作用域绑定外层，内部变量a不会到函数f体内取值，所以输出1，而不是2

参数可省略

JavaScript允许省略参数，被省略的参数为undefined

省略参数不改变函数的length属性值，因为length属性值只反映函数预期传入的参数个数

注意，不能省略靠前的参数，如果必须省略，则需要显式地传入undefined

arguments 对象

由于参数可省略且不定长，所以需要一种机制来读取函数体的所有参数，包括省略的参数，于是就有了arguments 对象，它包含了函数在运行时的所有参数。这个对象只有在函数体内部才能使用

通过arguments对象的length属性，可以判断函数调用时到底带几个参数

arguments对象还有一个callee属性，返回它所对应的原函数

可以通过arguments.callee，达到调用函数自身的目的。这个属性在严格模式里面是禁用的，因此不建议使用

值传递与引用传递

原始数据类型（数值、布尔值、字符串、null、undefined）属于值传递（passes by value）

复合类型的值（数组、对象、其他函数）属于引用传递（pass by reference）

注意：如果函数内部修改的，不是参数对象的某个属性，而是替换掉整个参数，这时不会影响到原始值

空位和undefined

数组的空位是可读取的，返回的是undefined

数组的某个位置是空位，与某个位置是undefined，是不一样的。如果是空位，使用数组的forEach方法、for...in结构、以及Object.keys方法进行遍历，空位都会被跳过

但如果某个位置是undefined，遍历的时候就不会被跳过

理解：空位即没有元素，所以不会被遍历到；undefined显式写出，则表示这个元素的值是undefined，所以会被遍历到

但是，读取空位上的元素时又会返回undefined
