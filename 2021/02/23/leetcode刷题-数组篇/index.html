485. 最大连续1的个数

难度：简单

题意

给定一个二进制数组， 计算其中最大连续 1 的个数。

  • 输入的数组只包含 0 和 1 。
  • 输入数组的长度是正整数，且不超过 10,000。

示例

解法

方法一：一次遍历

从头到尾遍历一遍数组，遇到1就累加cnt变量，遇到0就先做一次记录，更新当前最大连续1的个数，再将cnt变量重置

注意点：遍历完毕后，还需要做一次记录的更新，这是为了处理全为1的情况

时间复杂度：O(n) 空间复杂度：O(1)

方法二：双指针法

双指针法，s记录1开始的位置，e记录1结束的位置。初始化s和e指向同一个位置元素，先确定s位置后，往后遍历找到e的位置，然后记录更新当前最大连续1的个数。下一次遍历的起点更新为e的下一个位置

时间复杂度：O(n) 空间复杂度：O(1)

283. 移动零

难度：简单

题意

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

  • 必须在原数组上操作，不能拷贝额外的数组。
  • 尽量减少操作次数。

示例

解法

方法一：两次遍历

只要从头到尾依次遍历，然后从头部开始依次填入非零的元素，就能保持非零元素的相对顺序。与此同时，记录非零元素的个数，处理完成后，只需再遍历一次，把剩下的位置全部插入0即可

时间复杂度：O(n) 空间复杂度：O(1)

方法二：双指针法

维护双指针i和j，使得j左边的都为非零数，i左边到j处都为0

当遇到非零数时，i和j指向同一个非零数（交换了个寂寞~），且同时前进一位。当遇到0时，j就暂且不动一直指向这个0，i往后直到找到一个非零数，来与这个0交换位置

时间复杂度：O(n) 空间复杂度：O(1)

27. 移除元素

难度：简单

题意

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并 原地 修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

说明:

为什么返回数值是整数，但输出的答案是数组呢?

请注意，输入数组是以「引用」方式传递的，这意味着在函数里修改输入数组对于调用者是可见的。

你可以想象内部操作如下:

  • 0 <= nums.length <= 100
  • 0 <= nums[i] <= 50
  • 0 <= val <= 100

解法

方法一：双指针法

思路和前面那题移动零差不多，只不过这题不用考虑数组中超出新长度后面的元素，相当于只用一次遍历即可，后面的元素不用管

时间复杂度：O(n) 空间复杂度：O(1)

方法二：双指针法的进一步优化

考虑一种情况，如果要移除的元素很少，用上面那种办法则需要很多次不必要的赋值操作，或者说是值覆盖操作

比如，输入nums = [1, 2, 2, 3, 3, 3]，val = 1。按照题意，我们完全可以把1和末尾的3交换一下位置，然后再把交换后的1删了，即变成nums = [3, 2, 2, 3, 3]，因为元素可为任意顺序，所以这完全符合题意。但是如果按方法一，则需要进行5次值覆盖操作，显然效率更低

时间复杂度：O(n) 空间复杂度：O(1)
